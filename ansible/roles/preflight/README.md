# Роль: preflight

## Назначение

Роль выполняет ранние проверки целевой VM до изменения системы. Ее задача - остановить playbook с понятной ошибкой, если VM не подходит для стенда Kyverno MVP.

## Что проверяется

- целевая OS относится к Debian family;
- дистрибутив - Debian или Ubuntu;
- установлен `sudo`;
- Ansible-пользователь может выполнять команды через privilege escalation;
- RAM не меньше `min_ram_mb`;
- CPU не меньше `min_cpu_count`;
- размер корневого диска `/` не меньше `min_disk_gb`.

## Входные переменные

Значения по умолчанию задаются в `ansible/roles/preflight/defaults/main.yml`:

```yaml
min_ram_mb: 7500
min_cpu_count: 2
min_disk_gb: 15
```

## Результат

Если проверки успешны, playbook продолжает выполнение следующих ролей. Если VM не подходит, роль завершает playbook через `ansible.builtin.fail` и выводит причину.

## Границы ответственности

Роль ничего не устанавливает и не меняет на VM. Она только валидирует окружение.
